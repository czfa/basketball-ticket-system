# 第五章 系统的详细设计与实现

## 5 系统的详细设计与实现

### 5.1 接口及类的设计与实现

#### 5.1.1 接口设计思想

根据系统总体设计的思路，本系统采用RESTful API接口设计规范，接口设计遵循以下原则：

1. **统一资源标识**：使用名词表示资源，如`/api/matches`表示赛事资源，`/api/orders`表示订单资源
2. **HTTP方法语义化**：GET用于查询，POST用于创建，PUT用于更新，DELETE用于删除
3. **统一响应格式**：所有接口返回统一的JSON格式，包含`success`、`message`、`data`等字段
4. **状态码规范**：使用标准HTTP状态码（200成功、400错误请求、401未授权、404未找到、500服务器错误）
5. **错误处理统一**：所有错误返回统一的错误格式，包含错误信息和错误码

#### 5.1.2 用户认证模块接口实现

##### 5.1.2.1 用户注册接口实现

**接口名称**：`POST /api/auth/register`

**接口功能**：用户注册，创建新用户账号

**设计思想**：
- 验证用户名和邮箱的唯一性，防止重复注册
- 使用BCrypt算法加密密码，保证密码安全
- 设置默认角色和状态，简化用户注册流程

**输入参数**（请求体，JSON格式）：
```json
{
  "username": "testuser",
  "password": "123456",
  "email": "test@example.com",
  "phone": "13800138000",
  "realName": "张三"
}
```

| 参数名 | 数据类型 | 是否必填 | 说明 |
|--------|---------|---------|------|
| username | String | 是 | 用户名，长度1-50字符，唯一 |
| password | String | 是 | 密码，长度6-20字符 |
| email | String | 否 | 邮箱地址 |
| phone | String | 否 | 手机号 |
| realName | String | 否 | 真实姓名 |

**输出参数**（响应体，JSON格式）：
```json
{
  "success": true,
  "message": "注册成功！"
}
```

| 参数名 | 数据类型 | 说明 |
|--------|---------|------|
| success | Boolean | 操作是否成功 |
| message | String | 提示信息 |

**实现代码**（AuthController）：
```java
@PostMapping("/register")
public ResponseEntity<?> register(@RequestBody User user) {
    Map<String, Object> response = new HashMap<>();
    try {
        userService.register(user);
        response.put("success", true);
        response.put("message", "注册成功！");
        return ResponseEntity.ok(response);
    } catch (RuntimeException e) {
        response.put("success", false);
        response.put("message", e.getMessage());
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(response);
    }
}
```

**UserService.register()方法实现**：

```java
/**
 * 用户注册逻辑
 * @param user 用户实体对象
 * @throws RuntimeException 用户名或邮箱已存在时抛出异常
 */
public void register(User user) {
    // 检查用户名是否已存在
    if (userMapper.selectOne(new QueryWrapper<User>().eq("username", user.getUsername())) != null) {
        throw new RuntimeException("用户名已存在");
    }
    
    // 检查邮箱是否已存在
    if (user.getEmail() != null && 
        userMapper.selectOne(new QueryWrapper<User>().eq("email", user.getEmail())) != null) {
        throw new RuntimeException("邮箱已存在");
    }

    // 密码加密存储（BCrypt）
    user.setPassword(passwordEncoder.encode(user.getPassword()));
    
    // 设置默认值
    LocalDateTime now = LocalDateTime.now();
    user.setRegisterTime(now);
    user.setRole(UserRole.USER);
    user.setStatus(UserStatus.ACTIVE);
    user.setUpdateTime(now);

    // 插入用户记录
    int result = userMapper.insert(user);
    if (result <= 0) {
        throw new RuntimeException("注册失败，请稍后重试");
    }
}
```

```json
}
```

**输出参数**（响应体，JSON格式）：
```json
{
  "success": true,
  "message": "登录成功",
  "user_info": {
    "userId": 1,
    "username": "testuser",
    "role": "USER"
  }

```

**实现代码**（AuthController）：
```java
@PostMapping("/login")
public ResponseEntity<?> login(@RequestBody Map<String, String> loginRequest, HttpSession session) {
    String username = loginRequest.get("username");
    String password = loginRequest.get("password");

    User user = userService.login(username, password);

    if (user != null) {
        // 设置Session
        session.setAttribute("user_id", user.getUserId());
        session.setAttribute("username", user.getUsername());
        session.setAttribute("role", user.getRole());

        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("message", "登录成功");
        response.put("user_info", Map.of(
            "userId", user.getUserId(),
            "username", user.getUsername(),
            "role", user.getRole()
        ));
        return ResponseEntity.ok(response);
    } else {
        Map<String, Object> response = new HashMap<>();
        response.put("success", false);
        response.put("message", "用户名或密码错误。");
        return ResponseEntity.status(HttpStatus.UNAUTHORIZED).body(response);
    }
}
```

**UserService.login()方法实现**：

```java
/**
 * 用户登录逻辑
 * @param username 用户名
 * @param password 密码
 * @return 登录成功返回用户对象，失败返回null
 */
public User login(String username, String password) {
    // 根据用户名查询用户
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.eq("username", username);
    User user = userMapper.selectOne(queryWrapper);
    
    if (user == null) {
        return null; // 用户不存在
    }
    
    // 验证密码（BCrypt）
    if (!passwordEncoder.matches(password, user.getPassword())) {
        return null; // 密码错误
    }
    
    // 检查账号状态
    if (!UserStatus.ACTIVE.equals(user.getStatus())) {
        return null; // 账号被禁用
    }
    
    // 更新最后登录时间
    user.setLastLoginTime(LocalDateTime.now());
    user.setUpdateTime(LocalDateTime.now());
    userMapper.updateById(user);
    
    return user;
}
```

##### 5.1.3.1 创建订单接口实现

**接口名称**：`POST /api/orders/create`

**接口功能**：创建订单并锁定座位（核心购票功能）

**设计思想**：
- 使用事务控制保证原子性：订单创建和座位锁定在同一事务中
- 防重复购票：业务层检查座位可用性，数据库层使用悲观锁
- 生成唯一订单号，便于订单追踪

**输入参数**（请求体，JSON格式）：
```json
{
  "matchId": 1,
  "seatIds": [1, 2, 3]
}
```

| 参数名 | 数据类型 | 是否必填 | 说明 |
|--------|---------|---------|------|
| matchId | Long | 是 | 赛事ID |
| seatIds | List<Long> | 是 | 座位ID列表 |
| userId | Long | 否 | 用户ID（可从Session获取） |

**输出参数**（响应体，JSON格式）：
```json
{
  "success": true,
  "orderId": 1,
  "orderNumber": "ORD1704067200000123",
  "totalAmount": 200.00,
  "seatCount": 2,
  "expireTime": "2025-01-01T10:30:00",
  "message": "订单创建成功，请在30分钟内完成支付"
}
```

**实现代码**（OrderController.createOrder()）：

```java
@PostMapping("/create")
public ResponseEntity<?> createOrder(@RequestBody Map<String, Object> request, HttpSession session) {
    try {
        // 从Session或请求体获取用户ID
        Long userId = (Long) session.getAttribute("user_id");
        if (userId == null) {
            Map<String, Object> response = new HashMap<>();
            response.put("success", false);
            response.put("message", "用户未登录，请先登录");
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).body(response);
        }

        // 验证用户是否存在
        User user = userService.getUserById(userId);
        if (user == null) {
            Map<String, Object> response = new HashMap<>();
            response.put("success", false);
            response.put("message", "用户不存在");
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).body(response);
        }

        // 获取请求参数
        Long matchId = Long.valueOf(request.get("matchId").toString());
        @SuppressWarnings("unchecked")
        List<Long> seatIds = new ArrayList<>();
        List<?> seatIdsRaw = (List<?>) request.get("seatIds");
        for (Object id : seatIdsRaw) {
            seatIds.add(((Number) id).longValue());
        }

        // 调用Service层创建订单（包含事务控制、防重复购票）
        Map<String, Object> result = orderService.createOrderAndPay(userId, matchId, seatIds);

        return ResponseEntity.ok(result);

    } catch (RuntimeException e) {
        Map<String, Object> response = new HashMap<>();
        response.put("success", false);
        response.put("message", e.getMessage());
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(response);
    }
}
```

**OrderService.createOrderAndPay()方法实现**：

```java
/**
 * 创建订单并自动支付（核心购票方法）
 * 使用事务控制保证原子性：订单创建 + 座位锁定 + 自动支付 + 票务生成
 * 
 * @param userId 用户ID
 * @param matchId 赛事ID
 * @param seatIds 座位ID列表
 * @return 包含订单信息的Map
 * @throws RuntimeException 座位不可用、赛事不存在、重复购票等情况抛出异常
 */
@Transactional(rollbackFor = Exception.class)
public Map<String, Object> createOrderAndPay(Long userId, Long matchId, List<Long> seatIds) {
    Map<String, Object> result = new HashMap<>();

    try {
        // 1. 验证赛事是否存在且可预订
        if (!matchService.isMatchBookable(matchId)) {
            throw new RuntimeException("该赛事当前不可预订");
        }

        // 2. 检查座位可用性（防重复购票 - 业务层检查）
        Map<String, Object> availabilityCheck = seatService.checkSeatsAvailability(seatIds);
        Boolean allAvailable = (Boolean) availabilityCheck.get("allAvailable");
        @SuppressWarnings("unchecked")
        List<Long> unavailableSeats = (List<Long>) availabilityCheck.get("unavailableSeats");

        if (!allAvailable) {
            throw new RuntimeException("部分座位不可用，请重新选择。不可用座位ID: " + unavailableSeats);
        }

        // 3. 计算总金额（从座位表获取价格）
        BigDecimal totalAmount = BigDecimal.ZERO;
        for (Long seatId : seatIds) {
            Seat seat = seatService.getSeatById(seatId);
            if (seat == null) {
                throw new RuntimeException("座位ID " + seatId + " 不存在");
            }
            // 验证座位属于该赛事
            if (!seat.getMatchId().equals(matchId)) {
                throw new RuntimeException("座位ID " + seatId + " 不属于该赛事");
            }
            totalAmount = totalAmount.add(seat.getPrice());
        }

        // 4. 生成订单号
        String orderNumber = "ORD" + System.currentTimeMillis() + (int)(Math.random() * 1000);

        // 5. 创建订单（直接设置为已支付状态，简化流程）
        Order order = new Order();
        order.setOrderNumber(orderNumber);
        order.setUserId(userId);
        order.setMatchId(matchId);
        order.setTotalAmount(totalAmount);
        order.setActualAmount(totalAmount);
        order.setSeatCount(seatIds.size());
        order.setOrderTime(LocalDateTime.now());
        order.setPayTime(LocalDateTime.now()); // 立即支付
        order.setStatus(OrderStatus.PAID); // 直接设置为已支付
        order.setCreateTime(LocalDateTime.now());
        order.setUpdateTime(LocalDateTime.now());

        // 保存订单
        boolean orderCreated = createOrder(order);
        if (!orderCreated) {
            throw new RuntimeException("创建订单失败");
        }

        // 6. 获取刚创建的订单ID
        Order savedOrder = getOrderByOrderNumber(orderNumber);
        if (savedOrder == null) {
            throw new RuntimeException("订单创建失败，无法获取订单ID");
        }
        Long orderId = savedOrder.getOrderId();

        // 7. 预订座位（锁定座位，防止重复购票 - 数据库层锁定）
        boolean seatsBooked = seatService.bookSeats(seatIds, orderId);
        if (!seatsBooked) {
            throw new RuntimeException("预订座位失败，可能座位已被其他用户预订");
        }

        // 8. 生成电子票
        boolean ticketsIssued = ticketsService.issueTicketsForOrder(orderId, seatIds);
        if (!ticketsIssued) {
            throw new RuntimeException("生成电子票失败");
        }

        // 9. 更新赛事可用座位数
        matchService.decreaseAvailableSeats(matchId, seatIds.size());

        result.put("success", true);
        result.put("orderId", orderId);
        result.put("orderNumber", orderNumber);
        result.put("totalAmount", totalAmount);
        result.put("seatCount", seatIds.size());
        result.put("message", "订单创建并支付成功，电子票已生成");

    } catch (RuntimeException e) {
        // 事务会自动回滚
        result.put("success", false);
        result.put("message", "创建订单失败: " + e.getMessage());
        throw e; // 重新抛出异常，触发事务回滚
    }

    return result;
}
```

```java
public Map<String, Object> checkSeatsAvailability(List<Long> seatIds) {
    Map<String, Object> result = new HashMap<>();
    List<Long> unavailableSeats = new ArrayList<>();
    
    for (Long seatId : seatIds) {
        Seat seat = getSeatById(seatId);
        if (seat == null || !seat.getIsAvailable() || seat.getIsBooked()) {
            unavailableSeats.add(seatId);
        }
    }
    
    result.put("allAvailable", unavailableSeats.isEmpty());
    result.put("unavailableSeats", unavailableSeats);
    return result;
}
```

2. **bookSeats方法**


**方法功能**：锁定座位，使用悲观锁防止并发问题

**输入参数**：
- `seatIds` (List<Long>)：座位ID列表

**返回值**：`Boolean`，是否锁定成功

**实现代码**：
```java
public boolean bookSeats(List<Long> seatIds, Long orderId) {
    for (Long seatId : seatIds) {
        // 使用条件更新，只有is_booked=0的座位才能被预订
        // 这是悲观锁的实现：UPDATE ... WHERE seat_id = ? AND is_booked = 0 AND is_available = 1
        QueryWrapper<Seat> wrapper = new QueryWrapper<>();
        wrapper.eq("seat_id", seatId)
               .eq("is_booked", 0)
               .eq("is_available", 1);
        
        Seat seat = new Seat();
        seat.setIsBooked(true);
        seat.setOrderId(orderId);
        seat.setUpdateTime(LocalDateTime.now());
        
        int updated = seatMapper.update(seat, wrapper);
        if (updated == 0) {
            // 座位已被其他用户预订，返回false
            return false;
        }
    }
    return true;
}
```

```
  │
  ▼
执行条件更新（悲观锁）
  │

  │
```

**方法签名**：

**方法功能**：为订单生成电子票，为每个座位生成一张票

**输入参数**：

- `seatIds` (List<Long>)：座位ID列表

```java
@Transactional(rollbackFor = Exception.class)
public boolean issueTicketsForOrder(Long orderId, List<Long> seatIds) {
    try {
        // 查询订单信息
        Order order = orderMapper.selectById(orderId);
        if (order == null) {
            throw new RuntimeException("订单不存在");
        }

        LocalDateTime now = LocalDateTime.now();

        // 为每个座位生成一张电子票
        for (Long seatId : seatIds) {
            Seat seat = seatMapper.selectById(seatId);
            if (seat == null) {
                throw new RuntimeException("座位ID " + seatId + " 不存在");
            }

            // 生成票号（TICK + orderId + seatId + 时间戳）
            String ticketNumber = "TICK" + orderId + "-" + seatId + "-" + System.currentTimeMillis();

            // 创建电子票
            Tickets ticket = new Tickets();
            ticket.setTicketNumber(ticketNumber);
            ticket.setOrderId(orderId);
            ticket.setSeatId(seatId);
            ticket.setStatus(TicketStatus.VALID);
            ticket.setCheckStatus(CheckStatus.UNCHECKED);
            ticket.setIssueTime(now);
            ticket.setCreateTime(now);
            ticket.setUpdateTime(now);

            // 插入电子票记录
            int result = ticketsMapper.insert(ticket);
            if (result <= 0) {
                throw new RuntimeException("生成电子票失败");
            }
        }

        return true;
    } catch (Exception e) {
        throw new RuntimeException("批量生成电子票失败: " + e.getMessage(), e);
    }
}
```

#### 5.2.3 算法与计算

##### 5.2.3.1 密码加密算法

**算法应用**：BCrypt密码加密算法

**算法原理**：
- BCrypt是一种单向哈希算法，基于Blowfish加密算法
- 算法自动生成随机盐值，每次加密结果不同
- 计算成本可调（strength参数），默认强度为10

**实现代码**：
```java
// 配置BCryptPasswordEncoder
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder(10); // 强度为10
}

// 加密密码
String encodedPassword = passwordEncoder.encode(plainPassword);
// 结果示例：$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

// 验证密码
boolean matches = passwordEncoder.matches(plainPassword, encodedPassword);
```

**算法优势**：
- 安全性高：单向哈希，无法还原明文密码
- 自动加盐：每次加密结果不同，防止彩虹表攻击
- 计算成本可调：可以根据硬件性能调整强度

##### 5.2.3.2 订单总金额计算

**计算算法**：

**算法描述**：遍历所选座位列表，累加每个座位的价格，得到订单总金额

**实现代码**：
```java
BigDecimal totalAmount = BigDecimal.ZERO;
for (Long seatId : seatIds) {
    Seat seat = seatService.getSeatById(seatId);
    if (seat == null) {
        throw new RuntimeException("座位ID " + seatId + " 不存在");
    }
    totalAmount = totalAmount.add(seat.getPrice());
}
```

**算法说明**：
- 使用`BigDecimal`类型，避免浮点数精度问题
- 时间复杂度：O(n)，n为座位数量
- 空间复杂度：O(1)

**计算示例**：
```
座位ID列表：[1, 2, 3]
座位1价格：100.00
座位2价格：150.00
座位3价格：200.00

计算过程：
totalAmount = 0
totalAmount = 0 + 100.00 = 100.00
totalAmount = 100.00 + 150.00 = 250.00
totalAmount = 250.00 + 200.00 = 450.00

结果：450.00
```

##### 5.2.3.3 订单号生成算法

**生成算法**：

**算法描述**：使用时间戳和随机数生成唯一订单号

**实现代码**：
```java
String orderNumber = "ORD" + System.currentTimeMillis() + (int)(Math.random() * 1000);
```

**算法说明**：
- 前缀：`ORD`（Order的缩写）
- 时间戳：`System.currentTimeMillis()`（毫秒级，13位数字）
- 随机数：0-999的随机整数（3位数字）
- 总长度：3 + 13 + 3 = 19位字符
- 示例：`ORD1704067200000123`

**算法优点**：
- 基本保证唯一性（时间戳唯一，随机数增加区分度）
- 实现简单，无需额外配置

**算法缺点**：
- 高并发场景下可能重复（同一毫秒内可能生成相同随机数）
- 建议优化：使用雪花算法（Snowflake ID）或数据库序列
