## 二、如何使用这些内容

### 2.1 撰写论文的步骤

#### 第一步：阅读参考文档
1. 阅读学校提供的《本科毕业论文（设计）撰写规范》
2. 使用《广州华商学院毕业论文模板》作为论文模板
3. 参考《陈宇涵-开题报告》的结构

#### 第二步：整理内容
1. 打开各章节的详细内容文档
2. 根据学校要求和实际情况，调整内容
3. 将内容复制到Word文档中，使用学校模板格式

#### 第三步：添加图表
1. 打开`系统架构图和ER图.md`
2. 使用Mermaid在线编辑器（https://mermaid.live/）将图表导出为图片
3. 将图片插入到Word文档的相应位置
4. 或者直接使用文本版的图表

#### 第四步：完善细节
1. 根据实际情况，补充代码截图
2. 添加系统运行截图
3. 完善参考文献
4. 检查格式是否符合学校要求

### 2.2 图表使用方法

#### 方法一：使用Mermaid在线编辑器

1. 访问 https://mermaid.live/
2. 复制`系统架构图和ER图.md`中的Mermaid代码
3. 粘贴到编辑器中
4. 点击"Download PNG"或"Download SVG"导出图片
5. 将图片插入到Word文档中

#### 方法二：使用VS Code插件

1. 在VS Code中安装"Mermaid Preview"插件
2. 打开包含Mermaid代码的Markdown文件
3. 预览图表
4. 右键导出为图片

#### 方法三：直接使用文本版

1. 复制`系统架构图和ER图.md`中的文本版图表
2. 使用等宽字体（如Courier New）格式化
3. 直接插入到Word文档中

### 2.3 代码使用方法

1. 各章节中的代码示例可以直接使用
2. 建议根据实际情况，从项目中截图关键代码
3. 代码要有适当的注释说明
4. 代码格式要规范，符合编程规范

### 2.4 测试用例使用方法

1. `论文详细内容-第六章-系统测试.md`中包含了21个测试用例
2. 可以根据实际情况，补充更多测试用例
3. 测试用例表格可以直接复制到Word文档中
4. 建议添加系统运行截图作为测试证据

## 四、注意事项

### 4.1 格式要求

1. **严格按照学校模板格式**：
   - 字体、字号、行距
   - 页眉页脚
   - 目录格式
   - 图表格式

2. **图表要求**：
   - 所有图表要有标题和编号
   - 图表要在正文中引用
   - 图表格式要统一

3. **代码要求**：
   - 代码要有注释
   - 代码格式要规范
   - 关键代码要有说明

### 4.2 内容要求

1. **真实性**：
   - 所有功能描述要基于实际实现
   - 代码要与论文描述一致
   - 测试结果要真实

2. **完整性**：
   - 各章节内容要完整
   - 不能遗漏重要内容
   - 逻辑要清晰

3. **准确性**：
   - 技术描述要准确
   - 不能有错误
   - 数据要准确

### 4.3 查重要求

1. **用自己的语言描述**：
   - 不要直接复制网络内容
   - 对参考内容进行改写
   - 代码注释要自己写

2. **避免重复**：
   - 检查论文中是否有重复内容
   - 对重复部分进行修改

3. **使用查重工具**：
   - 使用学校指定的查重系统
   - 查重率要符合要求（一般<30%）

## 六、常见问题

### Q1: 如何将Mermaid图表插入到Word文档中？

**A**: 
1. 访问 https://mermaid.live/
2. 粘贴Mermaid代码
3. 点击"Download PNG"导出图片
4. 在Word中插入图片

### Q2: 代码截图从哪里获取？

**A**: 
1. 从项目源代码中截图
2. 使用IDE的代码截图功能
3. 或者直接复制代码到Word中，使用代码格式

### Q3: 测试用例需要全部使用吗？

**A**: 
- 可以根据实际情况选择使用
- 建议至少包含核心功能的测试用例
- 可以补充更多测试用例

### Q4: 如何确保论文格式正确？

**A**: 
1. 严格按照学校模板格式
2. 使用Word的样式功能
3. 目录自动生成
4. 图表使用题注功能

### Q5: 查重率太高怎么办？

**A**: 
1. 用自己的语言重新描述
2. 对参考内容进行改写
3. 代码注释要自己写
4. 避免直接复制网络内容
