# HikariPool 初始化异常排查指南

## 问题症状
```
HikariPool-1 - Exception during pool initialization.
```

## 可能原因和解决方案

### 1. MySQL 服务未启动 ⭐ 最常见
**检查方法**：
- Windows: 打开"服务"，查找 "MySQL" 或 "MySQL80"，确保状态为"正在运行"
- 命令行: `net start MySQL` 或 `net start MySQL80`

**解决方案**：
- 启动 MySQL 服务
- 或在服务管理中设置为自动启动

---

### 2. 数据库不存在
**检查方法**：
```sql
-- 使用 MySQL 客户端登录
mysql -u root -p

-- 查看所有数据库
SHOW DATABASES;

-- 检查是否存在 basketball_ticket_system
```

**解决方案**：
如果数据库不存在，需要创建：
```sql
CREATE DATABASE basketball_ticket_system 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

然后导入 docs/数据库结构与数据.md 中的 SQL 语句。

---

### 3. 用户名或密码错误
**检查方法**：
```sql
-- 尝试使用配置的用户名和密码登录
mysql -u root -p
-- 输入密码: 123456
```

**解决方案**：
- 如果无法登录，说明密码错误，需要：
  1. 重置 MySQL root 密码
  2. 或者修改 application.properties 中的密码为正确密码
  3. 或者创建新用户并授予权限：
     ```sql
     CREATE USER 'ticket_user'@'localhost' IDENTIFIED BY 'your_password';
     GRANT ALL PRIVILEGES ON basketball_ticket_system.* TO 'ticket_user'@'localhost';
     FLUSH PRIVILEGES;
     ```

---

### 4. 端口号不正确
**检查方法**：
```sql
-- 查看 MySQL 监听的端口
SHOW VARIABLES LIKE 'port';
```

或者在 MySQL 配置文件中查看：
- Windows: `C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`
- 查找 `port=3306`

**解决方案**：
- 如果 MySQL 使用其他端口，修改 application.properties 中的端口号
- 或者修改 MySQL 配置使用 3306 端口

---

### 5. MySQL 驱动版本不兼容
**当前配置**：
- Spring Boot 版本: 3.1.6
- MySQL 驱动: mysql-connector-java 8.0.33

**解决方案**（如果需要）：
更新为新的驱动（Spring Boot 3.x 推荐）：
```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

但 mysql-connector-java 8.0.33 应该也可以工作。

---

### 6. 防火墙阻止连接
**检查方法**：
- 确保 Windows 防火墙允许 MySQL 的连接
- 检查是否有其他防火墙软件阻止

---

### 7. 连接URL格式问题
**当前URL**：
```
jdbc:mysql://localhost:3306/basketball_ticket_system?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
```

**已修复**：添加了 `&allowPublicKeyRetrieval=true` 参数，这有助于解决某些认证问题。

---

### 8. 查看完整错误信息
**重要**：HikariPool 的错误通常只是表面错误，真正的原因在更详细的堆栈信息中。

**检查方法**：
1. 查看完整的控制台日志
2. 查找 "Caused by" 关键字
3. 常见的真实错误信息可能是：
   - `Access denied for user 'root'@'localhost'` - 用户名或密码错误
   - `Unknown database 'basketball_ticket_system'` - 数据库不存在
   - `Communications link failure` - MySQL 服务未启动
   - `Connection refused` - 端口错误或服务未启动

---

## 快速诊断步骤

1. **检查 MySQL 服务是否运行**
   ```powershell
   # PowerShell
   Get-Service | Where-Object {$_.DisplayName -like "*MySQL*"}
   ```

2. **测试数据库连接**
   ```sql
   mysql -u root -p123456 -h localhost -P 3306 basketball_ticket_system
   ```
   如果这个命令成功，说明数据库配置是正确的。

3. **查看详细日志**
   - 查看应用启动时的完整控制台输出
   - 查找 "Caused by" 后面的错误信息

4. **检查数据库是否存在**
   ```sql
   SHOW DATABASES LIKE 'basketball_ticket_system';
   ```
