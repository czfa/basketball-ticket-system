# 毕业论文撰写指南

## 一、撰写步骤

### 第一步：理解参考文档要求

根据您提供的参考文档，建议按以下顺序阅读：

1. **《本科毕业论文（设计）撰写规范（含样张）》** - 了解学校对论文格式、结构、字数的具体要求
2. **《广州华商学院毕业论文模板（不带批注版）》** - 使用学校提供的模板，确保格式符合要求
3. **《毕业设计说明书撰写指南》** - 了解毕业设计说明书的撰写要点
4. **《陈宇涵-开题报告》** - 参考开题报告的结构，确保论文内容与开题报告一致

### 第二步：整理项目资料

#### 2.1 项目概述
- **项目名称**：基于Spring Boot和Vue的篮球票务销售系统
- **技术栈**：
  - 后端：Spring Boot + MyBatis-Plus + MySQL
  - 前端：Vue 3 + Vue Router + Axios
  - 安全：Spring Security（BCrypt密码加密）

#### 2.2 核心功能模块
1. **用户端**：用户注册登录、赛事浏览、在线选座、购票下单、订单管理
2. **管理员端**：管理员登录、赛事管理、座位管理、订单统计

#### 2.3 项目文档清单
- `docs/第一阶段-数据库结构推断报告.md` - 数据库设计
- `docs/第二阶段-核心业务功能实现总结.md` - 后端业务逻辑
- `docs/第六阶段-前端页面实现总结.md` - 前端页面实现
- `docs/数据库结构与数据.md` - 数据库详细结构
- `docs/前端页面设计与实现说明.md` - 前端设计说明
- `代码检查报告.md` - 代码质量检查

### 第三步：确定论文结构

1. 研究背景与意义

4. 论文组织结构

2. Vue.js框架
3. MyBatis-Plus ORM框架
4. MySQL数据库
5. 前后端分离架构

#### 第三章：系统需求分析
1. 系统功能需求
   - 用户端功能需求

   - 性能需求
   - 安全性需求
   - 可用性需求

### 第四步：撰写各章节内容

#### 4.1 第一章：绪论

**研究背景与意义**：
- 传统票务销售方式的局限性
- 在线票务系统的优势
- 篮球赛事票务市场的需求

**研究内容与目标**：
- 设计并实现一个完整的篮球票务销售系统
- 实现用户在线选座、购票、订单管理功能
- 实现管理员后台管理功能
- 确保系统安全性和数据一致性

**参考内容**：可参考开题报告中的研究背景部分

**技术介绍要点**：

- 在本系统中的应用场景

## 三、论文撰写时间规划

### 建议时间安排（以8周为例）

**第1-2周**：阅读参考文档，整理项目资料，确定论文结构
- 阅读学校提供的撰写规范和模板
- 整理项目代码和文档
- 绘制系统架构图、ER图

**第3-4周**：撰写前四章（绪论、技术介绍、需求分析、系统设计）
- 第一章：绪论
- 第二章：相关技术介绍
- 第三章：系统需求分析
- 第四章：系统设计

**第5-6周**：撰写第五章（系统实现）和第六章（系统测试）
- 第五章：系统实现（后端、前端、数据库）
- 第六章：系统测试（功能测试、性能测试）

**第7周**：撰写第七章（总结与展望），完善论文
- 第七章：总结与展望
- 完善各章节细节
- 添加图表、代码示例

**第8周**：论文检查与修改
- 格式检查
- 内容检查
- 查重检查
- 根据导师意见修改

## 四、常见问题与解决方案

### 4.1 如何避免论文内容空洞？

**解决方案**：
- 结合项目实际代码进行说明
- 展示关键代码片段
- 提供系统运行截图
- 详细描述技术实现细节

### 4.2 如何提高论文技术深度？

**解决方案**：
- 深入分析关键技术实现（如事务控制、防重复购票）
- 说明技术选型理由
- 分析系统性能优化方案
- 讨论系统安全性设计

### 4.3 如何确保论文格式正确？

**解决方案**：
- 严格按照学校模板格式
- 使用Word的样式功能
- 目录自动生成
- 图表使用题注功能

### 4.4 如何降低查重率？

**解决方案**：
- 用自己的语言描述技术内容
- 代码注释要自己写
- 避免直接复制网络内容
- 对参考内容进行改写

## 五、论文撰写技巧

### 5.1 写作技巧

1. **逻辑清晰**：各章节之间要有逻辑关系，前后呼应
2. **重点突出**：核心功能和技术难点要详细描述
3. **图文并茂**：适当使用图表，提高可读性
4. **代码规范**：展示的代码要有注释，格式规范

### 5.2 技术描述技巧

1. **技术选型理由**：说明为什么选择这个技术
2. **实现细节**：详细描述关键功能的实现过程
3. **问题解决**：说明遇到的技术难点及解决方案
4. **性能优化**：说明系统的性能优化措施

### 5.3 图表制作技巧

1. **系统架构图**：清晰展示系统层次结构
2. **ER图**：准确反映数据库表关系
3. **流程图**：详细描述业务流程
4. **界面截图**：使用实际运行截图，标注关键功能
