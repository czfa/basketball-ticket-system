# 第二章 相关技术介绍

## 2.1 Spring Boot框架

### 2.1.1 Spring Boot简介

Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化Spring应用的初始搭建以及开发过程。Spring Boot基于Spring框架，通过自动配置和起步依赖，大大简化了Spring应用的开发。

### 2.1.2 Spring Boot核心特性

1. **自动配置（Auto Configuration）**：
   - Spring Boot能够根据类路径中的jar包、类，自动配置Spring应用
   - 开发者无需手动配置XML文件，大大减少了配置工作

2. **起步依赖（Starter Dependencies）**：
   - Spring Boot提供了一系列starter依赖，如spring-boot-starter-web、spring-boot-starter-security等
   - 通过引入starter依赖，可以快速集成所需功能，无需手动管理依赖版本

3. **内嵌服务器**：
   - Spring Boot内置了Tomcat、Jetty等Web服务器
   - 可以直接运行jar包启动应用，无需部署到外部服务器

4. **生产就绪特性**：
   - 提供健康检查、指标监控、外部化配置等生产环境所需的功能

### 2.1.3 在本系统中的应用

在本系统中，Spring Boot主要用于：

1. **Web服务开发**：使用spring-boot-starter-web快速搭建RESTful API服务
2. **安全框架集成**：使用spring-boot-starter-security实现用户认证和授权
3. **数据库集成**：集成MyBatis-Plus进行数据库操作
4. **配置管理**：使用application.properties进行系统配置

**技术优势**：
- 快速开发：通过自动配置和起步依赖，快速搭建项目框架
- 简化配置：减少XML配置，使用Java配置和注解
- 易于部署：内嵌服务器，可直接运行jar包

## 2.2 Vue.js框架

### 2.2.1 Vue.js简介

Vue.js是一套用于构建用户界面的渐进式JavaScript框架。Vue.js的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。

### 2.2.2 Vue.js核心特性

1. **响应式数据绑定**：
   - Vue.js使用数据劫持和发布-订阅模式实现响应式数据绑定
   - 当数据发生变化时，视图自动更新

2. **组件化开发**：
   - Vue.js支持组件化开发，可以将页面拆分为多个可复用的组件
   - 提高代码的可维护性和可复用性

3. **虚拟DOM**：
   - Vue.js使用虚拟DOM技术，提高渲染性能
   - 通过diff算法，只更新需要更新的DOM节点

4. **单文件组件**：
   - 支持.vue单文件组件，将模板、脚本和样式组织在一个文件中
   - 提高代码的组织性和可维护性

### 2.2.3 在本系统中的应用

在本系统中，Vue.js主要用于：

1. **用户界面构建**：使用Vue 3的Composition API构建用户界面
2. **路由管理**：使用Vue Router实现单页面应用的路由管理
3. **状态管理**：使用localStorage和Vuex管理应用状态
4. **API调用**：使用Axios进行HTTP请求，与后端API交互

**技术优势**：
- 学习曲线平缓：语法简洁，易于学习和使用
- 性能优秀：虚拟DOM和响应式系统提供良好的性能
- 生态丰富：拥有丰富的插件和工具支持

## 2.3 MyBatis-Plus ORM框架

### 2.3.1 MyBatis-Plus简介

MyBatis-Plus是一个MyBatis的增强工具，在MyBatis的基础上只做增强不做改变，为简化开发、提高效率而生。

### 2.3.2 MyBatis-Plus核心特性

1. **无侵入**：
   - 只做增强不做改变，引入它不会对现有工程产生影响
   - 可以随时切换回MyBatis

2. **损耗小**：
   - 启动即会自动注入基本CRUD，性能基本无损耗
   - 直接面向对象操作

3. **强大的CRUD操作**：
   - 内置通用Mapper、通用Service，仅需少量配置即可实现单表大部分CRUD操作
   - 支持Lambda形式调用，通过函数式编程优雅地编写代码

4. **支持Lambda形式调用**：
   - 通过Lambda表达式，避免字段名写错的问题

5. **支持主键自动生成**：
   - 支持多种主键策略，包括自增、UUID、雪花算法等

6. **支持ActiveRecord模式**：
   - 支持ActiveRecord形式调用，实体类只需继承Model类即可进行强大的CRUD操作

### 2.3.3 在本系统中的应用

在本系统中，MyBatis-Plus主要用于：

1. **数据库操作简化**：使用BaseMapper提供的通用方法，减少重复代码
2. **条件构造器**：使用QueryWrapper构建复杂的查询条件
3. **分页查询**：使用Page对象进行分页查询（需配置分页插件）
4. **自动填充**：使用MetaObjectHandler自动填充创建时间和更新时间

**技术优势**：
- 减少代码量：自动生成CRUD方法，减少重复代码
- 提高开发效率：条件构造器简化复杂查询
- 易于维护：代码结构清晰，易于理解和维护

## 2.4 MySQL数据库

### 2.4.1 MySQL简介

MySQL是一个关系型数据库管理系统，由瑞典MySQL AB公司开发，目前属于Oracle公司。MySQL是最流行的关系型数据库管理系统之一，在Web应用方面，MySQL是最好的RDBMS（Relational Database Management System，关系数据库管理系统）应用软件之一。

### 2.4.2 MySQL核心特性

1. **开源免费**：
   - MySQL是开源软件，可以免费使用
   - 拥有活跃的社区支持

2. **性能优秀**：
   - MySQL具有优秀的性能，能够处理大量数据
   - 支持多种存储引擎，如InnoDB、MyISAM等

3. **易于使用**：
   - SQL语法简单，易于学习和使用
   - 提供了丰富的管理工具

4. **可扩展性强**：
   - 支持主从复制、读写分离等扩展方案
   - 支持分库分表等高级特性

### 2.4.3 在本系统中的应用

在本系统中，MySQL主要用于：

1. **数据存储**：存储用户信息、赛事信息、订单信息等业务数据
2. **事务支持**：使用InnoDB存储引擎，支持事务处理，保证数据一致性
3. **索引优化**：为关键字段建立索引，提高查询性能
4. **外键约束**：使用外键约束保证数据完整性

**技术优势**：
- 成熟稳定：经过长期发展，技术成熟稳定
- 性能优秀：能够满足本系统的性能需求
- 易于维护：提供了丰富的管理工具

## 2.5 前后端分离架构

### 2.5.1 前后端分离架构简介

前后端分离架构是一种Web应用架构模式，将前端和后端的开发分离，前端负责用户界面和交互，后端负责业务逻辑和数据存储。

### 2.5.2 前后端分离架构特点

1. **职责分离**：
   - 前端专注于用户界面和用户体验
   - 后端专注于业务逻辑和数据处理

2. **技术栈独立**：
   - 前端可以使用任何前端框架（Vue、React、Angular等）
   - 后端可以使用任何后端框架（Spring Boot、Django、Express等）

3. **接口通信**：
   - 前后端通过RESTful API进行通信
   - 使用JSON格式进行数据交换

4. **部署独立**：
   - 前端和后端可以独立部署
   - 可以分别进行扩展和优化

### 2.5.3 在本系统中的应用

在本系统中，前后端分离架构的应用如下：

1. **前端**：使用Vue.js构建单页面应用，通过Axios调用后端API
2. **后端**：使用Spring Boot提供RESTful API服务
3. **通信协议**：使用HTTP/HTTPS协议，JSON格式进行数据交换
4. **跨域处理**：使用CORS配置解决跨域问题

**技术优势**：
- 开发效率高：前后端可以并行开发，提高开发效率
- 易于维护：前后端代码分离，易于维护和扩展
- 技术选型灵活：前后端可以选择最适合的技术栈
- 用户体验好：前端可以快速响应用户操作，提供流畅的用户体验
