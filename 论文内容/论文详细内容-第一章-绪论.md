# 第一章 绪论

## 1.1 研究背景与意义

### 1.1.1 研究背景

随着互联网技术的快速发展和体育产业的蓬勃兴起，传统的票务销售方式已经无法满足现代消费者的需求。传统的票务销售主要存在以下问题：

1. **购票效率低下**：传统购票方式需要消费者前往售票点排队购票，耗费大量时间和精力，特别是在热门赛事开售时，往往出现排长队、一票难求的情况。

2. **座位选择不直观**：传统购票方式中，消费者无法直观地看到场馆座位分布情况，只能通过座位号进行选择，难以判断座位的实际位置和视野效果。

3. **信息不透明**：消费者难以实时了解座位的可用情况、价格信息等，容易错过心仪的座位。

4. **管理成本高**：对于票务管理方而言，传统方式需要大量人力进行售票、检票等工作，管理成本高，且容易出现人为错误。

5. **数据统计困难**：传统方式难以对销售数据进行实时统计和分析，不利于制定营销策略和优化运营。

随着移动互联网的普及和Web技术的成熟，在线票务系统逐渐成为票务销售的主流方式。在线票务系统通过Web技术实现了票务信息的数字化管理，为消费者提供了便捷的购票体验，同时也为管理方提供了高效的管理工具。

### 1.1.2 研究意义

本研究旨在设计并实现一个基于Spring Boot和Vue的篮球票务销售系统，具有以下重要意义：

1. **理论意义**：
   - 探索前后端分离架构在实际项目中的应用，验证该架构在Web应用开发中的优势
   - 研究事务控制、并发控制等关键技术在高并发场景下的实现方案
   - 为类似系统的开发提供参考和借鉴

2. **实践意义**：
   - **提高购票效率**：用户可以通过Web浏览器随时随地浏览赛事信息、选择座位、完成购票，无需前往售票点，大大提高了购票效率
   - **改善用户体验**：提供直观的在线选座功能，用户可以实时查看座位分布和可用情况，选择心仪的座位
   - **降低管理成本**：系统自动化处理大部分票务管理工作，减少人工干预，降低管理成本
   - **数据统计分析**：系统自动记录和统计销售数据，为管理方提供数据支持，便于制定营销策略
   - **提高数据安全性**：采用密码加密、会话管理等安全机制，保障用户数据安全

3. **技术意义**：
   - 验证Spring Boot框架在快速开发Web应用中的优势
   - 验证Vue.js框架在构建用户友好界面方面的能力
   - 探索MyBatis-Plus在简化数据库操作方面的应用
   - 研究前后端分离架构下的系统设计和实现方法

## 1.2 国内外研究现状

### 1.2.1 国外研究现状

国外在线票务系统起步较早，技术相对成熟。主要特点包括：

1. **技术架构先进**：国外大型票务平台（如Ticketmaster、StubHub等）普遍采用微服务架构，系统可扩展性强，能够应对高并发访问。

2. **功能完善**：除了基本的购票功能外，还提供座位推荐、价格动态调整、转售等功能，用户体验较好。

3. **安全机制完善**：采用多种安全技术，如HTTPS加密传输、支付安全认证、防刷票机制等，保障系统安全。

4. **移动端支持**：提供移动App和响应式Web应用，支持多平台访问。

### 1.2.2 国内研究现状

国内在线票务系统发展迅速，主要特点包括：

1. **市场规模大**：随着体育产业和娱乐产业的发展，国内票务市场规模不断扩大，涌现出大麦网、猫眼电影、淘票票等大型票务平台。

2. **技术不断创新**：国内票务平台在技术方面不断创新，如采用分布式架构、引入人工智能推荐算法等。

3. **用户体验优化**：注重用户体验优化，提供在线选座、电子票、二维码检票等功能。

4. **移动支付普及**：与支付宝、微信支付等移动支付平台深度集成，支付流程便捷。

### 1.2.3 现有系统分析

虽然现有票务系统功能较为完善，但仍存在一些不足：

1. **系统复杂度高**：大型票务系统架构复杂，对于中小型赛事或场馆而言，部署和维护成本较高。

2. **定制化程度低**：通用票务系统难以满足特定业务需求，定制化开发成本高。

3. **技术门槛高**：对于初学者而言，理解大型系统的架构和实现较为困难。

因此，开发一个结构清晰、功能完整、易于理解和维护的票务系统，对于学习和研究具有重要价值。

## 1.3 研究内容与目标

### 1.3.1 研究内容

本研究的主要内容包括：

1. **系统需求分析**：
   - 分析用户端和管理员端的功能需求
   - 分析系统的非功能需求（性能、安全性、可用性等）
   - 绘制用例图，明确系统功能边界

2. **系统设计**：
   - 设计系统总体架构，采用前后端分离架构
   - 设计数据库结构，包括用户表、赛事表、座位表、订单表等
   - 设计各功能模块，包括用户认证模块、赛事管理模块、选座购票模块等

3. **系统实现**：
   - 使用Spring Boot框架实现后端服务
   - 使用Vue.js框架实现前端界面
   - 使用MyBatis-Plus实现数据库操作
   - 实现关键业务逻辑，如事务控制、防重复购票等

4. **系统测试**：
   - 进行功能测试，验证各功能模块是否正常工作
   - 进行性能测试，验证系统在高并发场景下的表现
   - 进行安全性测试，验证系统的安全机制

### 1.3.2 研究目标

本研究的目标包括：

1. **完成系统设计与实现**：
   - 设计并实现一个功能完整的篮球票务销售系统
   - 实现用户注册登录、赛事浏览、在线选座、购票下单、订单管理等核心功能
   - 实现管理员后台管理功能，包括赛事管理、座位管理、订单统计等

2. **确保系统安全性和数据一致性**：
   - 实现密码加密存储（BCrypt）
   - 实现会话管理，保障用户登录安全
   - 实现事务控制，确保数据一致性
   - 实现防重复购票机制，避免并发问题

3. **提供良好的用户体验**：
   - 设计直观友好的用户界面
   - 实现实时座位状态更新
   - 提供流畅的选座购票流程

4. **验证技术选型的合理性**：
   - 验证Spring Boot在快速开发中的优势
   - 验证Vue.js在构建用户界面方面的能力
   - 验证前后端分离架构的可行性

## 1.4 论文组织结构

本论文共分为七章，各章内容安排如下：

**第一章 绪论**：介绍研究背景与意义、国内外研究现状、研究内容与目标，以及论文组织结构。

**第二章 相关技术介绍**：介绍本系统所使用的主要技术，包括Spring Boot框架、Vue.js框架、MyBatis-Plus ORM框架、MySQL数据库，以及前后端分离架构。

**第三章 系统需求分析**：分析系统的功能需求和非功能需求，绘制用例图，明确系统功能边界。

**第四章 系统设计**：设计系统总体架构、数据库结构，以及各功能模块的设计方案。

**第五章 系统实现**：详细介绍系统的实现过程，包括后端实现、前端实现和数据库实现，展示关键代码。

**第六章 系统测试**：介绍系统的测试过程，包括功能测试、性能测试和安全性测试，分析测试结果。

**第七章 总结与展望**：总结本研究的成果，分析系统的特点与创新点，指出不足与改进方向，展望未来工作。
