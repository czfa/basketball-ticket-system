# 前端页面使用说明

## 一、项目依赖问题说明

### 1.1 Element Plus 依赖问题

部分页面（`UserLogin.vue`, `UserRegister.vue`, `MatchList.vue`）使用了 Element Plus 组件，但项目 `package.json` 中没有安装 Element Plus。

**解决方案有两种**:

#### 方案一：安装 Element Plus（推荐）

如果希望使用 Element Plus UI 框架，可以安装：

```bash
cd Web/qd
npm install element-plus
npm install @element-plus/icons-vue
```

然后在 `main.js` 中引入：

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'
import axios from 'axios'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import * as ElementPlusIconsVue from '@element-plus/icons-vue'

const app = createApp(App)
app.use(store)
app.use(router)
app.use(ElementPlus)

// 注册所有图标
for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}

app.mount('#app')

axios.defaults.baseURL = 'http://localhost:8088'
axios.defaults.withCredentials = true
```

#### 方案二：使用原生 HTML/CSS（适合毕业设计）

如果不希望引入额外的UI框架，可以将使用 Element Plus 的页面改为原生 HTML/CSS 实现。

需要修改的文件：
- `src/views/user/UserLogin.vue` - 改为原生表单实现
- `src/views/user/UserRegister.vue` - 改为原生表单实现
- `src/views/user/MatchList.vue` - 改为原生HTML/CSS实现

可以参考 `UserLogin.vue`（原有组件）和 `MatchDetail.vue` 的实现风格。

### 1.2 其他依赖

其他依赖已在 `package.json` 中：
- Vue 3
- Vue Router 4
- Vuex 4
- Axios

## 二、运行步骤

### 2.1 安装依赖

```bash
cd Web/qd
npm install
```

如果使用 Element Plus：
```bash
npm install element-plus @element-plus/icons-vue
```

### 2.2 启动开发服务器

```bash
npm run serve
```

前端服务将运行在 `http://localhost:8080`（或Vue CLI自动分配的端口）

### 2.3 确保后端服务运行

后端服务需要运行在 `http://localhost:8088`，如果端口不同，需要修改：
- `src/api/index.js` 中的 `API_BASE_URL`
- `src/main.js` 中的 `axios.defaults.baseURL`

## 五、常见问题

### 5.1 跨域问题

前端已配置 `axios.defaults.withCredentials = true`，确保后端允许跨域并设置相应的 CORS 头。

后端需要配置：
```java
@CrossOrigin(origins = "http://localhost:8080", allowCredentials = "true")
```

### 5.2 Session Cookie 问题

前端使用 Session Cookie 进行会话管理，需要确保：
- 后端设置了 Session Cookie
- 前端请求携带 Cookie（已配置 `withCredentials: true`）
- 浏览器允许第三方 Cookie（开发环境可能需要）

### 5.3 路由404问题

如果访问路由出现404，检查：
- 路由配置是否正确
- 路由守卫是否阻止了访问
- 浏览器控制台是否有错误信息

### 5.4 接口调用失败

如果接口调用失败，检查：
- 后端服务是否运行
- 接口地址是否正确（`src/api/index.js`）
- 请求参数是否正确
- 浏览器控制台网络请求是否有错误

## 六、代码规范说明

### 6.1 文件命名

- 页面组件：使用 PascalCase，如 `MatchList.vue`
- API 文件：使用 camelCase，如 `index.js`
- 样式文件：使用 kebab-case（如果需要单独的样式文件）

### 6.2 代码风格

- 使用 Vue 3 Composition API
- 使用 `<script setup>` 或 `setup()` 函数
- 使用 TypeScript（可选）
- 使用 ES6+ 语法
- 添加必要的注释（中文）

### 6.3 样式规范

- 使用 Scoped CSS（`<style scoped>`）
- 使用语义化的类名
- 避免使用内联样式
- 使用 CSS 变量（如果需要主题切换）

## 八、毕业设计论文使用

### 8.1 前端功能描述（论文可用）

本系统前端实现了完整的篮球票务销售功能，包括用户端和管理员端两个部分。

**用户端功能**:
- 用户注册与登录：实现了用户账号的注册和登录认证功能
- 赛事浏览：支持赛事列表展示、详情查看、搜索和状态筛选，采用分页方式展示
- 在线选座：实现了可视化座位图，支持区域筛选、座位选择和可用性实时检查，核心功能为选座页面
- 订单管理：支持创建订单、支付订单、取消订单、查询订单历史等功能
- 个人中心：个人信息管理和密码修改（待实现）

**管理员端功能**:
- 管理员登录：独立的管理员认证系统
- 数据概览：展示系统统计数据（总赛事数、总订单数、总销售额、已售票数）
- 赛事管理：赛事的增删改查、状态更新、票价设置（待实现）
- 座位管理：批量生成座位、单个座位编辑、批量删除（待实现）
- 订单统计：订单查询、销售统计、赛事销售分析（待实现）

**技术特点**:
- 采用 Vue 3 框架，使用 Composition API 提高代码可维护性
- 前后端分离架构，通过 RESTful API 进行通信
- 使用 Session Cookie 进行会话管理，确保安全性
- 实现了路由守卫，区分用户和管理员访问权限
- 响应式设计，支持不同屏幕尺寸的设备访问
- 使用原生 CSS 实现样式，代码简洁，符合毕业设计要求

### 8.2 典型页面功能流程（论文可用）

**选座购票流程**:

1. **赛事浏览阶段**: 用户在赛事列表页浏览所有可购买的篮球赛事，可以通过搜索栏搜索特定赛事，也可以通过状态筛选（全部/即将开始/进行中/已结束）筛选赛事。

2. **赛事详情查看**: 用户点击感兴趣的赛事卡片，进入赛事详情页，查看赛事的完整信息，包括对阵双方、比赛时间、比赛场馆、基础票价、可用座位数等。

3. **进入选座页面**: 用户点击"立即选座购票"按钮，系统首先检查用户登录状态。如果用户未登录，系统会跳转到登录页，并记录原本要访问的页面。用户登录成功后，系统会自动跳转到选座页面。如果用户已登录，则直接进入选座页面。

4. **座位选择过程**: 
   - 系统加载该赛事的座位列表，按区域（Zone）组织展示
   - 座位状态用不同颜色区分：绿色表示可选（未售），蓝色表示已选（用户已选择），灰色表示已售（已被其他用户购买），白色表示不可用（系统禁用）
   - 用户可以通过区域筛选下拉框选择特定区域，只显示该区域的座位
   - 用户点击可选座位（绿色），该座位会变成蓝色（已选），并显示在右侧的已选座位列表中，同时显示座位信息和价格
   - 用户再次点击已选座位（蓝色），可以从已选列表中移除该座位
   - 已售座位（灰色）和不可用座位（白色）不可点击
   - 已选座位列表显示已选座位的详细信息（区域、座位号、价格），用户可以点击"移除"按钮移除已选座位
   - 右侧结算区显示已选座位数量和总金额

5. **订单确认**: 用户选择完座位后，点击"确认选座并下单"按钮。系统会再次调用后端接口检查座位的可用性（防止并发抢座）。如果所有座位都可用，系统会将选座信息保存到 `sessionStorage` 中，然后跳转到订单确认页。如果部分座位不可用（可能被其他用户抢先购买），系统会提示用户"部分座位已被其他用户预订，请重新选择"，并刷新座位状态，清空已选座位列表，让用户重新选择。

6. **创建订单**: 在订单确认页，用户可以看到赛事的详细信息、已选座位列表、订单金额。用户需要选择支付方式（支付宝/微信支付/银行卡），然后点击"确认下单"按钮。系统调用后端接口创建订单。如果订单创建成功，系统会清除 `sessionStorage` 中的选座信息，并跳转到订单详情页，显示订单信息。如果订单创建失败，系统会显示错误信息。

7. **订单管理**: 订单创建成功后，用户可以在"我的订单"页查看所有订单。订单列表支持按状态筛选（全部/待支付/已支付/已完成/已取消/已退款）。待支付订单显示过期时间，如果订单即将过期（30分钟内），过期时间会以红色高亮显示。待支付订单可以取消或立即支付。已支付订单显示支付时间和交易号。用户可以点击订单卡片查看订单详情。

### 8.3 前端技术栈说明（论文可用）

本系统前端采用以下技术栈：

- **Vue 3**: 采用最新的 Vue 3 框架，使用 Composition API 提高代码的可维护性和复用性
- **Vue Router 4**: 用于实现单页面应用（SPA）的路由管理，支持路由守卫实现权限控制
- **Vuex 4**: 用于全局状态管理（当前版本主要用于扩展，核心状态使用 localStorage）
- **Axios**: 用于发送 HTTP 请求，与后端 RESTful API 进行通信
- **原生 CSS**: 使用原生 CSS 实现样式，不使用第三方 UI 框架，代码简洁，便于维护和定制

前端采用前后端分离架构，通过 RESTful API 与后端进行通信。前端负责用户界面的展示和用户交互，后端负责业务逻辑处理和数据存储。这种架构的优点是前后端职责清晰，开发效率高，便于维护和扩展。
