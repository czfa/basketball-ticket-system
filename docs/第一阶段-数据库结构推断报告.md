### 2.1 用户表（user）

```sql
CREATE TABLE `user` (
  `user_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `password` VARCHAR(255) NOT NULL COMMENT '密码（加密存储）',
  `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
  `phone` VARCHAR(20) DEFAULT NULL COMMENT '手机号',
  `real_name` VARCHAR(50) DEFAULT NULL COMMENT '真实姓名',
  `id_card` VARCHAR(18) DEFAULT NULL COMMENT '身份证号',
  `role` VARCHAR(20) DEFAULT 'USER' COMMENT '用户角色（USER/ADMIN等）',
  `status` VARCHAR(20) DEFAULT 'ACTIVE' COMMENT '账号状态（ACTIVE/INACTIVE/BANNED）',
  `avatar_url` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
  `register_time` DATETIME DEFAULT NULL COMMENT '注册时间',
  `last_login_time` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `update_time` DATETIME DEFAULT NULL COMMENT '更新时间',
```

- `status`：账号状态，可能值：ACTIVE/INACTIVE/BANNED
- `deleted`：逻辑删除字段（MyBatis-Plus配置中有定义，但实体类未显式声明）

### 2.4 座位表（seat）

```sql
CREATE TABLE `seat` (
  `seat_id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '座位ID',
  `match_id` BIGINT NOT NULL COMMENT '赛事ID',
  `seat_number` VARCHAR(50) NOT NULL COMMENT '座位号（如：A区1排10座）',
  `seat_zone` VARCHAR(20) DEFAULT NULL COMMENT '区域（如：A区）',
  `seat_row` INT DEFAULT NULL COMMENT '排号',
  `seat_col` INT DEFAULT NULL COMMENT '列号',
  `seat_type` VARCHAR(20) DEFAULT 'STANDARD' COMMENT '座位类型（VIP/PREMIUM/STANDARD）',
  `price` DECIMAL(10,2) NOT NULL COMMENT '实际价格',
  `is_booked` TINYINT(1) DEFAULT 0 COMMENT '是否已预订（0-未预订，1-已预订）',
  `is_available` TINYINT(1) DEFAULT 1 COMMENT '是否可用（0-不可用，1-可用）',
  `order_id` BIGINT DEFAULT NULL COMMENT '订单ID（预订时关联）',
  `create_time` DATETIME DEFAULT NULL COMMENT '创建时间',
  `update_time` DATETIME DEFAULT NULL COMMENT '更新时间',
  `deleted` TINYINT(1) DEFAULT 0 COMMENT '逻辑删除标识',
  PRIMARY KEY (`seat_id`),
  KEY `idx_match_id` (`match_id`),
  KEY `idx_order_id` (`order_id`),
  KEY `idx_match_seat_number` (`match_id`, `seat_number`),
  KEY `idx_match_zone_row_col` (`match_id`, `seat_zone`, `seat_row`, `seat_col`),
  KEY `idx_available_booked` (`is_available`, `is_booked`),
  CONSTRAINT `fk_seat_match` FOREIGN KEY (`match_id`) REFERENCES `matchrecord` (`match_id`) ON DELETE CASCADE,
  CONSTRAINT `fk_seat_order` FOREIGN KEY (`order_id`) REFERENCES `orders` (`order_id`) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='座位表';
```

**关键字段说明**：
- `match_id`：外键，关联赛事表
- `order_id`：外键，关联订单表（可空，预订时填入）
- `is_booked`：是否已预订（购票时设为true）
- `is_available`：是否可用（管理员可手动设置不可用座位）

**索引设计理由**（高并发购票场景）：
- `idx_match_id`：按赛事查询座位（最常用）
- `idx_match_seat_number`：唯一性检查（同一赛事座位号不重复）
- `idx_match_zone_row_col`：复合索引，支持按区域、排、列查询和排序
- `idx_available_booked`：查询可用座位（`is_available=1 AND is_booked=0`）

**业务约束**：
- 同一赛事的`seat_number`应唯一（代码中通过`getSeatByNumber`方法检查）
- 座位被预订时，`is_booked=1`且`order_id`不为空
- 支付成功后，座位的`order_id`保持不变；退票时，`is_booked=0`且`order_id`设为NULL

```sql
  `pay_method` VARCHAR(20) DEFAULT NULL COMMENT '支付方式',

  PRIMARY KEY (`order_id`),
  UNIQUE KEY `uk_order_number` (`order_number`),
  KEY `idx_user_id` (`user_id`),

  CONSTRAINT `fk_order_match` FOREIGN KEY (`match_id`) REFERENCES `matchrecord` (`match_id`) ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

**关键字段说明**：
- `order_number`：唯一订单号（业务生成，格式：ORD + 时间戳）
- `status`：订单状态枚举值
  - PENDING：待支付（下单后30分钟内需支付）
  - PAID：已支付
  - CANCELLED：已取消（未支付超时或被用户取消）
  - REFUNDED：已退款
  - COMPLETED：已完成（已检票）
  - EXPIRED：已过期（待支付超时）
- `expire_time`：订单过期时间（下单后30分钟）

**索引设计理由**：
- `idx_user_id`：查询用户订单列表（高频查询）
- `idx_status`：按状态筛选订单（管理端统计）
- `idx_user_status`：复合索引，用户查看自己的特定状态订单（如：待支付订单）

**业务约束**：
- 订单编号唯一（代码中通过`getOrderByOrderNumber`方法检查）
- 订单与赛事、用户外键约束（删除用户或赛事时，订单应被保留，故使用RESTRICT）

## 四、索引设计建议（高并发购票场景）

### 4.1 核心索引（已在上表结构中标注）

1. **seat表**：
   - `idx_match_id`：查询某赛事的座位（最频繁）
   - `idx_available_booked`：查询可用座位（`is_available=1 AND is_booked=0`）
   - `idx_match_zone_row_col`：按区域、排、列排序展示座位图

2. **orders表**：
   - `idx_user_id`：用户查询自己的订单
   - `idx_user_status`：用户查询特定状态订单（如待支付）

3. **matchrecord表**：
   - `idx_match_time`：按时间排序展示赛事列表

## 五、业务必需但可能缺失的字段

### 5.1 已识别的问题

1. **seat表的`order_id`字段**：
   - ✅ 已存在（代码中`Seat`实体有`orderId`字段）
   - 用于锁定座位，但外键约束可能导致删除订单时座位无法释放
   - **建议**：删除订单时，先将`seat.order_id`设为NULL，再删除订单

2. **逻辑删除字段`deleted`**：
   - ⚠️ **问题**：`MybatisPlusConfig`中配置了`deleted`字段自动填充，但所有Entity类中都没有`@TableLogic`注解和`deleted`字段
   - **推断**：数据库中应有`deleted`字段，但实体类未定义，导致逻辑删除功能未生效
   - **建议**：要么在实体类中添加`deleted`字段并使用`@TableLogic`注解，要么删除配置中的逻辑删除相关代码

3. **订单编号生成策略**：
   - ✅ 代码中使用`"ORD" + System.currentTimeMillis()`生成
   - **风险**：并发下单可能产生重复订单号
   - **建议**：使用雪花算法或数据库序列生成

### 5.2 可能缺失的字段（需确认）

1. **order表的`version`字段**（乐观锁）：
   - `MybatisPlusConfig`中配置了乐观锁插件，但`Order`实体类中没有`version`字段
   - **推断**：数据库可能没有`version`字段，导致乐观锁失效

2. **matchrecord表的`start_sale_time`和`end_sale_time`字段**：
   - 当前只有`match_time`（比赛时间），缺少售票时间范围
   - **建议**：如需控制售票时间窗口，应添加此字段
