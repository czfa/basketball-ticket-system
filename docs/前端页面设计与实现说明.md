├── views/                  # 页面组件
│   ├── user/              # 用户端页面
│   │   ├── UserLogin.vue         # 用户登录页
│   │   ├── UserRegister.vue      # 用户注册页
│   │   ├── MatchList.vue         # 赛事列表页
│   │   ├── MatchDetail.vue       # 赛事详情页
│   │   ├── SeatSelection.vue     # 在线选座页（核心）
│   │   ├── OrderConfirm.vue      # 订单确认页
│   │   ├── MyOrders.vue          # 我的订单页
│   │   ├── OrderDetail.vue       # 订单详情页
│   │   └── UserProfile.vue       # 个人中心页（待实现）

│       ├── AdminDashboard.vue    # 后台首页（数据概览）

### 1.2 技术栈

- **框架**: Vue 3 (Composition API)
- **路由**: Vue Router 4
- **状态管理**: Vuex 4
- **HTTP请求**: Axios
- **样式**: 原生CSS（不使用Element Plus等UI框架，保持毕业设计简洁风格）

## 二、路由结构

### 2.1 用户端路由

| 路径 | 名称 | 组件 | 说明 | 是否需要登录 |
|------|------|------|------|-------------|
| `/` | Home | - | 首页（重定向到赛事列表） | 否 |
| `/login` | UserLogin | UserLogin.vue | 用户登录 | 否 |
| `/register` | UserRegister | UserRegister.vue | 用户注册 | 否 |
| `/matches` | MatchList | MatchList.vue | 赛事列表（支持分页、搜索、筛选） | 否 |
| `/matches/:matchId` | MatchDetail | MatchDetail.vue | 赛事详情 | 否 |
| `/matches/:matchId/seats` | SeatSelection | SeatSelection.vue | 在线选座（核心页面） | 是 |
| `/orders/confirm` | OrderConfirm | OrderConfirm.vue | 订单确认 | 是 |
| `/orders` | MyOrders | MyOrders.vue | 我的订单列表 | 是 |
| `/orders/:orderId` | OrderDetail | OrderDetail.vue | 订单详情 | 是 |
| `/profile` | UserProfile | UserProfile.vue | 个人中心 | 是 |

### 2.2 管理员端路由

| 路径 | 名称 | 组件 | 说明 | 是否需要管理员权限 |
|------|------|------|------|------------------|
| `/admin/login` | AdminLogin | AdminLogin.vue | 管理员登录 | 否 |
| `/admin` | AdminDashboard | AdminDashboard.vue | 管理后台首页（数据概览） | 是 |
| `/admin/matches` | AdminMatchManagement | AdminMatchManagement.vue | 赛事管理（CRUD） | 是 |
| `/admin/seats/:matchId?` | AdminSeatManagement | AdminSeatManagement.vue | 座位管理（批量生成、维护） | 是 |
| `/admin/orders` | AdminOrderManagement | AdminOrderManagement.vue | 订单管理与统计 | 是 |

### 2.3 路由守卫

路由守卫实现了以下功能：

1. **用户登录检查**: 访问需要登录的页面时，未登录自动跳转到登录页
2. **管理员权限检查**: 访问管理员页面时，未登录或非管理员自动跳转到管理员登录页
3. **已登录用户**: 访问登录/注册页时，自动跳转到首页
4. **页面标题设置**: 根据路由meta信息自动设置页面标题

## 三、API接口配置

### 3.1 API配置文件 (`src/api/index.js`)

所有后端接口地址统一管理在 `src/api/index.js` 文件中，包括：

- **认证接口** (`authAPI`): 用户注册、登录、登出、个人信息管理
- **赛事接口** (`matchAPI`): 赛事列表、详情、搜索、热门赛事
- **座位接口** (`seatAPI`): 座位列表、可用座位、区域筛选、可用性检查
- **订单接口** (`orderAPI`): 创建订单、支付、取消、查询订单
- **管理员接口** (`adminAPI`): 管理员登录、赛事管理、座位管理、订单统计

### 3.2 接口调用示例

```javascript
// 用户登录
import { authAPI } from '@/api'
const response = await authAPI.login(username, password)

// 获取赛事列表
import { matchAPI } from '@/api'
const response = await matchAPI.getMatchList(1, 10, 'UPCOMING')

// 创建订单
import { orderAPI } from '@/api'
const response = await orderAPI.createOrder(matchId, seatIds)
```

## 四、核心页面功能说明

### 4.1 用户登录页 (`UserLogin.vue`)

**功能**: 用户登录认证

**接口调用**: 
- `POST /api/auth/login`

**关键逻辑**:
- 表单验证（用户名、密码）
- 登录成功后保存用户信息到 `localStorage`
- 跳转到原本要访问的页面或默认跳转到赛事列表

**注意事项**: 
- 使用 Session Cookie 进行会话管理
- 登录状态存储在 `localStorage` 中，键为 `user_info`

### 4.2 赛事列表页 (`MatchList.vue`)

**功能**: 展示所有可购买的篮球赛事，支持分页、搜索、状态筛选

**接口调用**: 
- `GET /api/matches/list` - 获取赛事列表（分页）
- `GET /api/matches/search` - 搜索赛事

**关键功能**:
- 分页展示赛事列表
- 支持按状态筛选（全部/即将开始/进行中/已结束）
- 支持关键词搜索（赛事名称、球队、场馆）
- 显示赛事基本信息（对阵、时间、场馆、票价、可用座位）
- 点击"立即购票"跳转到选座页面

**状态判断**:
- 只有状态为 `UPCOMING` 且有可用座位的赛事才能购票
- 其他状态的赛事显示"暂不可售"

### 4.3 赛事详情页 (`MatchDetail.vue`)

**功能**: 展示单场赛事的详细信息

**接口调用**: 
- `GET /api/matches/:matchId`

**关键信息展示**:
- 赛事名称、对阵双方
- 比赛时间、场馆
- 赛事状态、基础票价
- 座位情况（可用/总共）
- 赛事描述（如果有）

**操作按钮**:
- "立即选座购票" - 跳转到选座页面（需要登录）

### 4.4 在线选座页 (`SeatSelection.vue`) - **核心页面**

**功能**: 在线选择座位并加入购物车

**接口调用**: 
- `GET /api/matches/:matchId` - 获取赛事信息
- `GET /api/seats/match/:matchId` - 获取座位列表
- `POST /api/seats/check-availability` - 检查座位可用性

**关键功能**:
1. **座位图展示**:
   - 按区域（Zone）展示座位
   - 座位状态区分：可选（绿色）、已选（蓝色）、已售（灰色）、不可用（白色）
   - 显示座位号和价格

2. **座位选择**:
   - 点击可选座位，加入已选列表
   - 点击已选座位，从已选列表移除
   - 已售和不可用座位不可点击

3. **区域筛选**:
   - 支持按区域筛选座位

4. **已选座位管理**:
   - 显示已选座位列表（区域、座位号、价格）
   - 支持移除已选座位

5. **订单确认**:
   - 显示已选座位数量和总金额
   - 点击"确认选座并下单"时：
     - 再次检查座位可用性（防止并发抢座）
     - 如果所有座位可用，跳转到订单确认页
     - 如果有座位不可用，提示用户并刷新座位状态

**状态管理**:
- 座位状态根据后端返回的 `isAvailable` 和 `isBooked` 字段判断：
  - `isAvailable = false` → 不可用
  - `isAvailable = true && isBooked = false` → 可选
  - `isAvailable = true && isBooked = true` → 已售

**注意事项**:
- 选座信息存储在 `sessionStorage` 中，键为 `selected_seats`
- 下单前必须再次检查座位可用性，确保数据一致性

### 4.5 订单确认页 (`OrderConfirm.vue`)

**功能**: 确认订单信息并创建订单

**数据来源**: 
- 从 `sessionStorage` 获取选座信息

**接口调用**: 
- `POST /api/orders/create` - 创建订单

**关键信息展示**:
- 赛事信息（名称、对阵、时间、场馆）
- 已选座位列表（区域、座位号、类型、价格）
- 订单总额

**功能**:
- 选择支付方式（支付宝/微信/银行卡）
- 确认下单：调用创建订单接口
- 订单创建成功后，清除 `sessionStorage` 中的选座信息
- 跳转到订单详情页

### 4.6 我的订单页 (`MyOrders.vue`)

**功能**: 查询用户的所有订单

**接口调用**: 
- `GET /api/orders/my` - 获取用户订单列表（支持状态筛选）

**关键功能**:
- 支持按状态筛选订单（全部/待支付/已支付/已完成/已取消/已退款）
- 显示订单基本信息（订单号、下单时间、状态、金额、座位数量）
- 订单状态用不同颜色标识
- 待支付订单显示过期时间，即将过期（30分钟内）用红色高亮

**操作按钮**:
- 待支付订单：显示"取消订单"和"立即支付"按钮
- 所有订单：显示"查看详情"按钮

### 4.7 订单详情页 (`OrderDetail.vue`)

**功能**: 展示订单详细信息

**接口调用**: 
- `GET /api/orders/:orderId` - 获取订单详情
- `POST /api/orders/:orderId/cancel` - 取消订单
- `POST /api/orders/:orderId/pay` - 支付订单

**关键信息展示**:
- 订单基本信息（订单号、状态、下单时间、支付时间、过期时间）
- 订单金额

**操作按钮**:
- 待支付订单：显示"取消订单"和"立即支付"按钮

### 4.8 管理员登录页 (`AdminLogin.vue`)

**功能**: 管理员登录认证

**接口调用**: 
- `POST /api/admin/login`

**关键逻辑**:
- 管理员登录成功后保存管理员信息到 `localStorage`，键为 `admin_info`
- 跳转到管理后台首页

### 4.9 管理后台首页 (`AdminDashboard.vue`)

**功能**: 数据概览和导航

**接口调用**: 
- `GET /api/admin/statistics/summary` - 获取基础数据统计

**关键功能**:
- 显示数据统计卡片（总赛事数、总订单数、总销售额、已售票数）
- 提供导航菜单（数据概览、赛事管理、座位管理、订单管理）
- 退出登录功能

## 五、页面功能流程说明

### 5.1 选座购票流程

```
1. 用户浏览赛事列表
   ↓
2. 点击"立即购票"或"立即选座购票"
   ↓
3. 检查是否登录
   ├─ 未登录 → 跳转到登录页（记录原本要访问的页面）
   └─ 已登录 → 进入选座页面
   ↓
4. 选座页面加载赛事信息和座位列表
   ↓
5. 用户选择座位（点击座位）
   ├─ 可选座位 → 加入已选列表
   ├─ 已选座位 → 从已选列表移除
   └─ 已售/不可用 → 不可操作
   ↓
6. 点击"确认选座并下单"
   ↓
7. 再次检查座位可用性（防止并发）
   ├─ 所有座位可用 → 保存选座信息到sessionStorage → 跳转到订单确认页
   └─ 有座位不可用 → 提示用户 → 刷新座位状态 → 清空已选座位
   ↓
8. 订单确认页
   ├─ 展示赛事信息、已选座位、订单金额
   ├─ 选择支付方式
   └─ 点击"确认下单"
   ↓
9. 创建订单（调用后端接口）
   ├─ 成功 → 清除sessionStorage → 跳转到订单详情页
   └─ 失败 → 提示错误信息
   ↓
10. 订单详情页
    ├─ 显示订单信息
    ├─ 待支付订单：可取消或立即支付
    └─ 已支付订单：显示支付信息
```

### 5.2 订单支付流程

```
1. 用户在我的订单页或订单详情页点击"立即支付"
   ↓
2. 调用支付接口（模拟支付）
   ↓
3. 支付成功
   ├─ 更新订单状态为"已支付"
   ├─ 生成电子票
   ├─ 更新赛事可用座位数
   └─ 刷新页面显示最新状态
```

## 六、状态管理与数据持久化

### 6.1 用户状态管理

- **登录状态**: 存储在 `localStorage`，键为 `user_info`
- **管理员状态**: 存储在 `localStorage`，键为 `admin_info`
- **选座信息**: 存储在 `sessionStorage`，键为 `selected_seats`（临时数据，下单后清除）

### 6.2 状态同步

- 登录/登出时更新 `localStorage`
- 路由守卫检查 `localStorage` 中的登录状态
- 页面刷新后从 `localStorage` 恢复登录状态

## 七、错误处理与用户提示

### 7.1 错误处理策略

1. **网络错误**: 显示"网络连接失败，请检查网络"
2. **接口错误**: 显示后端返回的错误信息
3. **业务错误**: 
   - 座位已被预订 → 提示"部分座位已被其他用户预订，请重新选择"
   - 订单状态错误 → 提示相应的错误信息

### 7.2 用户提示方式

- 使用浏览器原生 `alert()` 进行错误提示（简洁，符合毕业设计要求）
- 关键操作（如删除、取消订单）使用 `confirm()` 确认

```bash
cd Web/qd
npm install
```

```bash
npm run serve
```

```bash
npm run build
```

构建产物将生成在 `dist/` 目录中

## 十一、毕业设计论文说明

### 11.1 前端页面功能说明（论文可用）

本系统前端采用 Vue 3 + Vue Router + Axios 技术栈，实现了完整的篮球票务销售系统的用户端和管理员端功能。

**用户端功能**:
- 用户注册与登录：支持用户账号注册和登录认证
- 赛事浏览：支持赛事列表展示、详情查看、搜索和状态筛选
- 在线选座：可视化座位图，支持区域筛选、座位选择和可用性实时检查
- 订单管理：创建订单、支付订单、取消订单、查询订单历史
- 个人中心：个人信息管理和密码修改

**管理员端功能**:
- 管理员登录：独立的管理员认证系统
- 赛事管理：赛事的增删改查、状态更新、票价设置
- 座位管理：批量生成座位、单个座位编辑、批量删除
- 订单统计：订单查询、销售统计、赛事销售分析

**技术特点**:
- 前后端分离架构，RESTful API 通信
- Session Cookie 会话管理
- 路由守卫实现权限控制
- 响应式设计，支持多端访问
- 原生CSS实现，代码简洁易维护

### 11.3 典型页面功能流程说明

**选座购票流程**（论文中可详细描述）:

1. **赛事浏览阶段**: 用户在赛事列表页浏览所有可购买的篮球赛事，可以通过搜索、状态筛选等方式查找感兴趣的赛事。

2. **赛事详情查看**: 点击赛事卡片进入详情页，查看赛事完整信息（对阵双方、比赛时间、场馆、票价、座位情况等）。

3. **进入选座页面**: 点击"立即选座购票"按钮，系统检查用户登录状态，未登录则跳转到登录页，已登录则进入选座页面。

4. **座位选择过程**: 
   - 系统加载该赛事的座位列表，按区域展示
   - 用户可以通过区域筛选查找特定区域的座位
   - 座位状态用不同颜色区分：绿色表示可选，蓝色表示已选，灰色表示已售，白色表示不可用
   - 用户点击可选座位，座位加入已选列表；再次点击已选座位，从列表中移除

5. **订单确认**: 用户选择完座位后，点击"确认选座并下单"，系统再次检查座位可用性（防止并发抢座），所有座位可用则跳转到订单确认页。

6. **创建订单**: 在订单确认页，用户确认赛事信息、已选座位、订单金额，选择支付方式，点击"确认下单"创建订单。

7. **订单管理**: 订单创建成功后，用户可以在"我的订单"页查看所有订单，待支付订单可以取消或立即支付，已支付订单显示订单详情。
